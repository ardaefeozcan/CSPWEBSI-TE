Big Idea 1 Creative Development

Video 3.1-3.4

  • Variables -- > A variable is an abstraction used to store values in a program
  • Data Storage -- > Mechanism to save data temporarily or permanently to use it somewhere else in code.
  • Data Types -- > Integers, Strings, Booleans.
  • Abstraction -- > Simplifying complex systems by focusing on important or required details.
  • Lists -- > Ordered collections of elements.
  • Functions -- > Blocks of reusable code that perform specific tasks. Mostly used in long code segments which are intended to be used more than one time.
  • Sequencing -- > Executing instructions(in other word code) in a specific order.
  • Iteration -- > Repeating steps (basically a loop structure) until a condition is met.

Video 3.5-3.8

  • Algorithm efficiency -- > It is how well an algorithm utilizes resources like time and memory, focusing on performance, optimization in order to be effective in problem solving.
  • Debugging -- > Identifying and fixing errors in code.
  • Encoding: Representing data in a specific format.
  • Boolean -- > A data type with two possible values: true or false.
  • And / Or / Not -- > Respectivly; logical operation where both should be true, atleast one should be true and invert of the value should taken.

Video 3.9- 3.12

  • While -- > A loop that repeats(iterates) while a condition is true.
  • For -- > A loop that repats(iterates) over a sequence or range.
  • Simulation -- > Creating a virtual model of a real-world system in order to simplify the process of writing code in oreder to solve a real-world problem.
  • Comments -- > Annotations in code to explain functionality. In swift they are made by putting "//" before the comment.
  • Documentation -- > Written explanations for understanding and using code. Very important as it helps other people to understand and optimeze you code according to their needs. Also it helps with remembering the purpose of various parts of code.
  • Clarity -- >Ensuring code and its purpose are understandable and clear. In other words making enough abstraction

Video 3.13-3.14

  • Logic -- > Correctness and reasoning in a program. Basically code can be diveded into 2 parts: Logic and Syntax. Logic is the purpose and syntax is the grammer of the code.
  • Libraries -- > Prewritten code modules for common tasks. These libaries conssit useful datasets and function for the coders. For an example SwiftUI is very important to deasign a user interface.
  • APIs -- > Interfaces for interacting with libraries or services.
